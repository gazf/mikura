## ADR-008: 権限モデル — パス + プリンシパル × アクション

**決定**: 「ユーザー / グループ」×「パス」×「read / write / admin」の細粒度認可。

**Deno KV スキーマ**:

```
["users", userId]                   → { id, name, email, groups }
["groups", groupId]                 → { id, name, members }
["permissions", path, type, id]     → { path, principal, access }
```

**アクセスチェック**:

- パスの親階層まで遡って判定
- ユーザー自身 + 所属グループの権限をチェック
- いずれかで許可があれば OK
